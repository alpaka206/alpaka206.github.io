앱 registry / shell config 구조

- shell.ts가 launcher 중심 registry 역할 수행
- LauncherEntry는 id, label, icon, subtitle, keywords, launch를 가짐
- launch.kind는 page, folder, browser, code, text-file, terminal 중 하나
- BROWSER_APPS는 browser shell에서 쓰는 URL과 메타를 정의
- CODE_WORKSPACES는 github1s 대상 owner, repo, branch, path를 정의
- DEFAULT_DESKTOP_ITEMS는 초기 바탕화면 아이콘 집합

window state 관리 방식

- Zustand 단일 store인 useDesktopStore가 windows 배열, activeWindowId, zIndexCounter 관리
- Pages 창은 단일 창 안에 여러 탭을 보관
- Text file, Terminal, Folder, Code, Browser는 개별 창으로 관리
- focusWindow 호출 시 z-index 상승
- toggleTaskbarItem은 active 상태면 minimize, minimized 상태면 focus 수행

launcher / search 동작 방식

- Start 메뉴 데이터 소스는 START_MENU_APPS
- 검색은 label, subtitle, keywords를 합친 문자열 기준 부분 일치 방식
- 추천 앱은 recentLauncherIds가 있으면 최근 실행 기준 사용
- recentLauncherIds가 없으면 START_MENU_RECOMMENDED_IDS 사용
- launcher 실행 시 recordLauncherUse 호출로 최근 실행 기록 갱신

wallpaper 저장 방식

- store: useDesktopPreferencesStore
- key: desktop_preferences
- field: wallpaperId
- quick panel과 Settings 페이지 모두 setWallpaper 호출

terminal command routing 방식

- 구현 위치: src/features/terminal-window/components/TerminalWindowBody.tsx
- 입력은 소문자와 공백 정규화 후 분기
- 지원 명령
  help
  clear
  exit
  open about
  open projects
  open resume
  open github
  open browser
  open settings
  open notepad
  wallpaper list
  wallpaper set <id>
- 실제 동작은 useDesktopStore의 openPage, openFolder, openTextFileWindow, closeWindow와 useDesktopPreferencesStore의 setWallpaper 호출로 연결
